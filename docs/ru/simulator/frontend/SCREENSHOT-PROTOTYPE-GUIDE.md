Этот документ отвечает на вопрос: **«на основе каких документов можно разработать скриншоты экранов, служащие прототипом функционала»**.

Функциональные требования, состояния, критерии приёмки, минимальный контракт данных, обязательные элементы HUD и панели управления — брать из:

- [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:1)

**Правило:** если есть расхождения между документами — верить SoT.

### Визуальный референс (look & feel)

Визуальная метафора, палитра, стилистика HUD/«космической карты», примеры компоновки — брать из:

- [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:1)

**Правило:** это визуальный референс; он не расширяет рамку Этапа 1.

### Продуктовый контекст (зачем это и где в продукте живёт)

Контекст «приложение‑симулятор», назначение и смысл визуализации — брать из:

- [docs/ru/simulator/specs/GENERAL-simulator-application.md](../specs/GENERAL-simulator-application.md:1)

**Правило:** использовать для терминов/подписей/ракурса «зачем», но не добавлять новые фичи в прототип без SoT.

## Минимальный набор экранов/сцен (Этап 1)

Ниже — минимальные скриншоты‑прототипы (экраны/состояния), которые должны покрывать функционал Этапа 1.

### 1) Главный экран симулятора с графом (Idle)

**Что должно быть на скриншоте (элементы):**

- Full-screen сцена с графом (Canvas) и поверх него HTML HUD.
- Фон «deep space» / звёздная пыль.
- Узлы двух типов (business/person) и связи между ними.
- Видимая панель управления (кнопки Tx/Clearing).

**Какие данные (поля) показать:**

- Тип узла должен читаться без текста (форма/цвет).

**Обязательные состояния/подсветки:**

- Нет выбранного узла.
- Нет активных эффектов Tx/Clearing.

**Привязка к источникам:**

- Функциональная рамка «граф + HUD + панель управления»: [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:54)
- Визуальная метафора/палитра: [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:83)

### 2) Состояние: выбранный узел + подсветка связей + затемнение остальных

**Что должно быть на скриншоте (элементы):**

- Один выбранный узел (явно выделен: glow/контур/яркость).
- Подсвечены только связи выбранного узла.
- Все прочие связи затемнены (очень низкая opacity).

**Какие данные (поля) показать:**

- Выбор узла должен быть очевиден визуально; идентификация узла может быть дополнена HUD карточкой (см. следующий пункт).

**Обязательные состояния/подсветки:**

- «Neighbors only» подсветка связей (обязательное поведение Этапа 1).

**Привязка к источникам:**

- Правила подсветки (обязательные): [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:410)
- Референс «фильтр связей выбранного узла»: [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:149)

### 3) HUD карточка участника (варианты business/person)

Нужно подготовить **два скриншота** карточки — для `business` и для `person`.

**Что должно быть на скриншоте (элементы):**

- Карточка в стиле HUD (glassmorphism/blur) поверх сцены.
- В карточке виден тип участника (иконка/лейбл/цвет), соответствующий узлу.

**Какие данные (поля) показать — минимум:**

- `name`
- `type` (business/person)
- `balance` (число со знаком)
- Опционально (если есть место): `trustLimitTotal` или аналогичный показатель.

**Обязательные состояния/подсветки:**

- Карточка соответствует выбранному узлу.
- Карточка не выглядит как «модальное окно» (сцена остаётся видимой).

**Привязка к источникам:**

- Минимальная модель данных для узла/карточки: [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:169)
- Визуальная «анатомия интерфейса» и пример карточки: [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:116)

### 4) Панель управления (кнопки Tx/Clearing)

**Что должно быть на скриншоте (элементы):**

- Панель управления в стиле game HUD.
- Кнопки (минимум):
  - `Single Tx`
  - `Run Clearing`

**Какие данные (поля) показать:**

- Текст кнопок (как в SoT).

**Обязательные состояния/подсветки:**

- Вариант A (idle): кнопки доступны.
- Вариант B (clearing running): `Run Clearing` disabled (и/или блокировка Tx — если принято блокировать Tx во время clearing).

**Привязка к источникам:**

- Требования к контролам: [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:296)
- Продуктовый контекст «экран карты сообщества»: [docs/ru/simulator/specs/GENERAL-simulator-application.md](../specs/GENERAL-simulator-application.md:6)

### 5) Состояние: анимация транзакции (Tx)

**Что должно быть на скриншоте (элементы):**

- Узлы A и B + связь между ними.
- Частица/комета (Tx) на траектории A → B, желательно со следом (trail).
- В момент Tx связь A–B визуально «active» (glow/яркость).

**Какие данные (поля) показать:**

- Не обязательно показывать суммы; достаточно читаемой анимации и направления.

**Обязательные состояния/подсветки:**

- Направление A → B читается.
- Есть видимый «импакт» на цели (вспышка) — можно отдельным кадром.

**Привязка к источникам:**

- Требования к Tx эффекту: [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:437)
- Визуальный референс частиц: [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:293)

### 6) Состояние: анимация клиринга (Clearing)

**Что должно быть на скриншоте (элементы):**

- Активный цикл/цепочка (минимум треугольник A–B–C–A) подсвечен.
- Встречные частицы/искры сходятся к midpoint.
- Вспышка/взрыв + шоквейв (кольцо) как ключевой маркер clearing.

**Какие данные (поля) показать:**

- На прототипе достаточно визуального события; суммы/точная математика не обязательны.

**Обязательные состояния/подсветки:**

- Во время clearing — видно disabled состояние на панели управления.
- Clearing визуально отличается от Tx по цвету/типу эффекта.

**Привязка к источникам:**

- Требования к clearing эффекту: [docs/ru/simulator/frontend/geo-simulator-phase1-tech-spec.md](geo-simulator-phase1-tech-spec.md:449)
- Референс «вспышка клиринга»: [docs/ru/simulator/frontend/Игровой интерфейс симулятора GEO.md](%D0%98%D0%B3%D1%80%D0%BE%D0%B2%D0%BE%D0%B9%20%D0%B8%D0%BD%D1%82%D0%B5%D1%80%D1%84%D0%B5%D0%B9%D1%81%20%D1%81%D0%B8%D0%BC%D1%83%D0%BB%D1%8F%D1%82%D0%BE%D1%80%D0%B0%20GEO.md:310)
