# GEO Hub Admin UI — Типографика и правила текстовых стилей

**Статус:** Нормативный документ (использовать при любых правках UI)

Цель: обеспечить единый, нехаотичный визуальный язык, где **заголовки**, **лейблы**, **значения** и **табличные заголовки** не конкурируют друг с другом.

---

## 1) Принципы

- Заголовок и название контрола **не равнозначны** по стилю.
- Название колонки и значение в строке **не равнозначны** по стилю.
- Заголовки колонок:
  - начинаются с заглавной буквы (Title case),
  - визуально сильнее значений.
- Не дублировать значения:
  - если рядом уже есть текстовое значение (например, `32%`), то у `el-progress` выключаем встроенный текст (`:show-text="false"`).
- Текст внутри виджетов по умолчанию — **Label/Hint**, а не Value:
  - роль **Value** (16px/700) использовать только для 1–2 ключевых KPI в карточке,
  - списки/расшифровки (например top1/top5/HHI, churn 7/30/90) должны быть в масштабе **Label/Hint**.

---

## 2) Роли (шкала)

Используем роли вместо “на глаз”:

- **Title** — заголовки секций/карточек.
- **Label** — лейблы полей, подписи, вторичный текст.
- **Value** — ключевые числа/результаты.
- **Table header** — заголовки колонок таблиц.
- **Hint** — пояснения/микротекст.

Рекомендуемая шкала (по умолчанию):
- Title: `13px / 600`
- Label: `12px / 500`, цвет `--el-text-color-secondary`
- Value: `16px / 700`
- Table header: `12–13px / 600`
- Hint: `12px`, вторичный цвет

---

## 3) Где это реализовано в коде

### 3.1 Глобальные токены

Файл: `admin-ui/src/style.css`
- CSS variables: `--geo-font-size-*`, `--geo-font-weight-*`

### 3.2 Глобальные классы (переиспользовать)

- `geoCard` — для карточек/страниц, чтобы заголовок имел роль Title.
- `geoTable` — для таблиц, чтобы заголовки колонок имели роль Table header.
- `geoLabel`, `geoHint`, `geoValue` — для ручной разметки, где Element Plus не задаёт роли автоматически.

---

## 4) Конвенции текста (copy rules)

### 4.0 Заголовки страниц/секций (layout)

- Заголовок секции (Title) и элементы управления (кнопки/селекты/фильтры) должны быть **на одной линии** (один «уровень»), как на страницах Trustlines/Participants.
- Паттерн: слева — Title, справа — controls (`.hdr` + `.hdr__right`), с переносом (wrap) только на узких экранах.
- Примечания/метаданные (например `Snapshot…`, `Updated…`) — отдельной строкой ниже заголовка, в роли **Hint** (`geoHint`), чтобы не конкурировать с Title.

### 4.1 Таблицы

- Использовать `geoTable` на `el-table` (включая страницы вроде Integrity, даже если таблица `border`).
- `geoTable` **нормализует типографику таблицы**:
  - общий размер шрифта в строках (body) = роль **Label**,
  - заголовки колонок (header) = роль **Table header** (жирнее, но не «больше в 2 раза»),
  - кликабельные элементы (link‑кнопки, ссылки) **наследуют размер шрифта**, сохраняя смысловые цвета.
- Не задавать локально (на страницах) произвольные `font-size` для таблиц — таблицы должны выглядеть одинаково на всех экранах.
- Заголовки колонок:
  - `Participant`, `Amount`, `Share`, `Status`, `Created at`, …
  - избегать технических сокращений в UI (`eq`, `tx_id`, `out_used`) — только если это общеизвестно и реально экономит место.
- Не допускать “сырого” ключа локализации в UI (`participant.pid`, `participant.displayName`, …). Если видите такое — это баг: заголовок должен быть человекочитаемым (например `participant.columns.pid`, `participant.drawer.displayName`).
- Текстовые колонки с потенциально длинными значениями:
  - использовать `TableCellEllipsis`/`show-overflow-tooltip`, чтобы таблица не «разъезжалась».
- Подсветка/ссылки — функционал, не декор:
  - цветовые акценты (например красный для узких мест/нарушений) и кликабельные PID/TxID **оставляем**;
  - типографика не должна ломать кликабельность и смысловые цвета.

### 4.1.1 Подсказки (tooltip)

- Если секция/карточка/таблица содержит термины, которые оператор может трактовать по‑разному (например `Net`, `Available`, `Bottleneck`) — добавляем короткую подсказку через `TooltipLabel`.
- Маркер `?` в `TooltipLabel` — **надстрочный** (superscript), компактный, не «в одну линию» с текстом.

### 4.2 Контролы

- Если нужен явный лейбл над полем — используем роль Label (`geoLabel`) и вторичный цвет.
- Placeholder не заменяет лейбл (placeholder — подсказка, не название поля).

### 4.3 Прогресс/проценты

- Если рядом выводим процент текстом — у `el-progress` ставим `:show-text="false"`.
